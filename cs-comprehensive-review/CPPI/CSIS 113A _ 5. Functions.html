Predefined Functions

In an earlier lesson, we said that C++ provides some functionality that you can use to make writing programs much easier. You may not realize it but you have already used some of these functions; Remember cin and cout? These are functions that are part of the standard library that give you the ability to read and write to the console. These two functions are a little odd because they are not called in the same way as most functions. This will become very clear to you in the advanced C++ class.

Some function basics

You should think of a function as a black box. I say this because you really don't need to know how the function works to use it. This is called encapsulation, which really is a term that means hiding the details from the user and it is a key concept in object oriented programming (Objects will be discussed down the road). Think of a function as a black box that does some work for you. There are many types of functions. Some of these function need to be supplied information before they can perform their job, some functions return a value back to, and finally some functions need some information and then will return an answer back to you. Lets look at one

In the above example, a double variable named root is declared, then the sqrt function is called with the value of 25 being passed to it. The sqrt function will then return 5 (which is the square root of 25) and assign it to the variable root.

When you want a function to perform a task you call it. You will also see the term referred to as invoke. In either case they mean the same thing; control of the program branches to where the function resides in memory, the function performs its task and returns back to where it was called from.

Important Point

It is important that you know how a function is called. A function like everything in a C++ program sits somewhere in memory. Where that is is unimportant. In fact, it is one of the great things about high level programming languages; we don't need to know where it is. To refer to the spot in memory you would just say sqrt (refers to the address). To have your program branch to where the function is you put the parentheses sqrt(). If the function requires information, it gets put in the parentheses sqrt(25) Some functions will need more than one piece of information and in that case you separate the values by a comma. When a function completes, it returns right back to where it was called from. In the sqrt example, it brings a value back with it. You can get the value that is returned by using the equal sign. In the above example, the program returned from the sqrt function with the value of 5 which was assigned to the variable root through the use of the = sign.

The cmath library

The sqrt function is part of the cmath library, and it must be included before you can call any of the functions that are part of it. The inclusion of cmath gives the compiler all of the definitions it needs for math functionality. You should know that the compiler needs to see a definition of a function before it can be called. It needs the definition so that it knows at compile time that you are calling a function correctly.

Function Arguments

Data passed to functions are called the functions arguments. You may also see arguments called functions parameters. Either way they refer to the same thing. Arguments or parameters that you use when you are calling a function are called the functions actual arguments (or actual parameters). Each of the arguments that are passed to a function must have a type. In the table above you will note that the fabs function has one actual argument and it must be of type double. The pow function has two arguments and they both must be of type double. You can pass parameters as either a literal or as a variable. In the examples in the table above, almost all arguments are doubles, in fact the pow function takes two doubles as parameters. You will see in the future that some functions will have mixed parameters, for instance a function could take as parameters, an int, a double, and a float, or just about any other combination you can think of. You will discover this better when writing your own functions.

What is this void thing?

Some functions don't return anything, an example from the table above is the exit function. Notice that the return type is void. void is a type in C++ and it is used to indicate nothing. In a face to face class I taught, I had a student really struggle with the concept of void. His question was; why do you need it if it means nothing? Well that is a tough question. It all comes down to the fact that the compiler needs a definition of the function before you can use it. To make all functions similar (meaning they all have a return type) void is used to indicate the function returns nothing.

In the coming lessons, we will discuss return types and parameters in much greater detail. This section was meant just to give you an idea of how to invoke functions that are part of the standard library.

Typically students ask how do I know what functions are out there for me to use? That is another difficult question and I can only tell you that it might be in your best interest to get a C++ complete reference. There have been several written that can be found at most bookstores.

In the future, I hope to have a page or two that is devoted to C++ functions for now, you are going to have to do a little research. My suggestion is, look at the standard library before you go off and try to write a function to do a task (that is unless you have specifically been told to do so in an assignment).

Programmer Defined Functions

The concept of functions can be found in other programming languages.  They are called methods, procedures, sub-routines just to name few but they all mean the same thing. You can think of functions as sub programs that allow you to organize your code in a logical fashion.   As a matter of fact, some languages call functions 'sub procedures'.

What are you really doing when you write a piece of software? The answer is that you are solving some sort of problem. The easiest way to tackle the problem is to break the large problem into several smaller problems and then tackle the smaller problems one at a time. In computer science, this is called stepwise refinement.  

After you have written a function and would like to use it, all you have to do is call (or invoke) it! The mechanics of this will become more clear in a little while but first there are a couple of things that you should know about functions. First, some functions need to be supplied some additional information before they can be used.  Other types of functions don't need any information, they just supply some information back to you.  Some functions do both. They need information to process and return some kind of value back to you. Finally, you should know that the most simple function doesn't take any additional information, and it doesn't produce any any value. It just performs a task.

A function is C++ basically consists of two sections



 

The function header consist of the return type, name of the function, parentheses, and optionally information that goes inside the parentheses. 

  • Return Type - All functions declarations must have a return type. If the function doesn't return anything then you need to use the keyword void. You can return any valid C++ type from a function. Later on you will learn that you can return you own custom made types.
  • Function Name - You can name a function anything that you like but you must also follow the same rules for naming variables.
  • Formal Parameters - The formal parameters (formal arguments) must be enclosed inside the parentheses following the function name. If you write a function that doesn't need any parameters then you just leave the parentheses empty. If you need multiple parameters sent to your function they must be separated by commas. Parameters are additional information that the function  needs to do it's task. For example, the  srand function uses a parameter as additional information to seed the random number generator.  A formal parameter then is the additional information needed by the function to allow it  to perform it's task. 
  • Curly Braces - The body of the function is always surrounded by open and closed curly braces. If you omit one or all of the braces then you will get a big syntax error.
  • Function Body - All of the code you want executed when the function is called (invoked) goes between the open and closed curly braces.

Code Review

 As in almost all programs we write, we have to include iostream and the std namespace. Notice that the next line of code looks just like the function header for the function helloWorld. This is called a function prototype, it is required or you will get a compilation error. Remember I said that the compiler needs to see a definition of any function before it can be used. Remember, this is why we include things like <iostream> and <cstdlib>. Since we have not gotten to the point where we create our own header files yet, we will just put the prototype above main. The compiler always goes through the file starting at the top and working down. This is why we putting the prototype (function definition ) first allows the compiler to see the definition before the function is used.

The first line of code in main is the call to the function called helloWorld. We can also say here that the function helloWorld is being invoked. Once the code reaches the function call control branches to the first line of code in the body of the  helloWorld function. The next line of code simply prints 'Hello World' to the screen. When the code reaches the last brace, control of the program branches back to where the function was called from (the first line of code within main in this case).

Common Mistake

A common mistake when creating the function prototype is forgetting to put the the semicolon at the end of it. If you leave the semicolon off of the prototype, the compiler will not know what it is you are trying to define because there are no curly braces to indicate the opening and closing of the function body.

The return statement

There are two ways that program can branch back to where it came from. The first is as you have already seen, when the closing brace of the function is reached. In this case the flow of the program will branch back to where it was called from. The second way is through the use of the return statement. You can place the return statement anywhere you want within a function and the program will branch back to where it came from.

Clarification of  Definitions

It is going to be said throughout this material that a function prototype is a definition. Someone may say and  has said that a function prototype is not a function definition. A function definition is the body of the function itself.  I would not disagree with that but when I say that a prototype is a definition I mean that a prototype defines the inputs and outputs to a function and this information must be seen by the compiler before the function can be used.   

Put another way, a prototype is a definition to the compiler to give it information about the function. This is also true with structs and classes (which will be seen soon). A class or a struct  for that matter is a blueprint for what gets built in memory. In other words, it defines  the number of bytes among other things that gets built in memory when an instance of it is created. 
  

What is important is that these definitions do not occupy space in memory. It is hoped that my use of the word definition when used in this context is clear. Of course I could go to the a thesaurus and find a different word to be used for definition. Maybe the term "statement of meaning" would be more appropriate.

Value Producing Functions

 

As the name implies a value producing function is a function that produces a value. Typically this is the answer to a specific problem. For instance, if you create a function called square the value produced is the square of the number given to it. 

 

Here is what the pieces of the square function look like. At this point it doesn't do anything but it is best to understand what each piece is.

 

If you recall the function header is the first line in the function. It consists of:

  • Return type - The type of data the function produces (int, char, float, double, bool, etc). 
  • Function Name - This can be almost anything but it should be descriptive. The name square indicates that this function will square something. 
  • Formal Parameters - This is the additional information the function needs to perform it's task.  The number to square in this case. These are called formal because they are formally stated. In other words the type of data to work on goes in front of the variable name. This really is a variable declaration. 

 

As with simple functions that function body is surrounded by curly braces. All of the code needed to perform the function's task go inside the open an closed curly brace. 

Before getting into passing information to a function we first need to look at the mechanism by which values are produced by functions. This mechanism is actually quite simple. In the function header you need to specify the type of data the method is producing and then in the method body simply return the value.

Passing Parameters

 

For the most part when you create functions you will need to provide additional information to the function so that it can perform it's task. For instance, if you were to create a function to square a number you will  probably want to provide the number to be squared. 


Information passed to functions is called arguments or parameters. You will often see these names used interchangeably but note that they both mean passing information to a function. Here, I will primarily use the word parameters.

Parameters as it turns out can be classified as two types:

  1. Formal Parameters
  2. Actual Parameters

Formal parameters are called formal because they are formally stated. This means that you need to specify the data type the parameter will hold. Formal parameters are found in the function header inside the parentheses.

The actual parameters are placed inside the parentheses when you invoke the method. Because they are not formal you do not need to provide a data type. You simply need to place the variable, value, or expression inside the parentheses.

Code Review

This simple function simply checks to see what value was passed to it. If x is greater than 5 then the function returns the value of x squared. If is less than 5, it returns the value of x cubed. The key thing here is to notice the use of the return statement.

 

Important Point

This function also takes as a formal parameter an integer value called x. This is actually like declaring an int variable inside the body of the function. In this case, it takes on a special meaning because it receives a copy of a value from where it was called. In the main function a variable called someValue is declared. It gets a value stored in it from whatever is typed in at the keyboard. When the function squareOrCube is called, a copy of the value that is stored at someValue gets sent to the function squareOrCube and stored in the newly created variable called x. Then, depending on whether the value of x is greater than 5 or not the square or the cube of the value of x is returned back to where the function was called from. Notice also that the return value is being stored back in the variable someValue. This is perfectly legal and sometimes preferred because it allows you to reuse a variable. This means that you will use less memory because you are declaring less variables.

 

How Parameters Are Passed

Parameters in C++ are passed by value. What passing by value means is that a copy of what is in the variable that is the actual parameter is pushed onto the stack. The code branches to the function where the copied values are then popped  off the stack and into the variables that are the formal parameters. Some beginning students tend to think that the names of the parameters both formal and actual need to be the same. What you need to realize is that the variable names really have very little to do with things. What is important to note is that a copy of what is in the actual parameter is taken and placed into the variables that are the formal parameters.

Scope Rules


When you write functions, you should try to make them as portable as possible. What I mean by portable is that you should be able to copy them from one program to another or put them in a library so that they can be used in any future program that you may write. Some of the functions in the previous section violated this rule but those functions were mainly meant to be examples. 

A function should be a self contained unit, kind of like a separate program that lies within a big program .This means that all of the variables that are declared within a function belong to that function.

It is important to realize that variables residing in different scope can have the same name. It is also important to know which one you are accessing. The inside variable will always have precedence (the local variable has precedence over the global of the same name).

The Stack and the Heap

When your program gets built, the compiler will set aside some memory called the stack The stack is a data structure that is like a stack of plates having basically two functions, push and pop. Values are pushed on to (added), and popped (removed) from the top of the stack. This kind of data structure is known as a LIFO (Last In First Out) data structure. The reason this is important, is because this is where local variables live. When a function is a called and variables are declared in the function, they are declared on the stack. When a function returns, the variables for the function are popped off the stack. This means that they are destroyed. In this way, C++ memory is like an accordion ( at least for local variables) it grows as variables are declared and it shrinks as they are discarded.

Global variables on the other hand are declared in a memory location called the heap. This is where all global and dynamic memory is stored. Global variable exist for the life of the program so they will consume your programs resources until your program stops running.

Avoid Use Of Global Variables

Looking at things, most beginning programmers fall in love with global variables. Why pass variables to functions as arguments if you don't have to? Well global variables are bad for several reasons. As mentioned above, global variables sit around eating up program resources. They also make it impossible to have your functions be loosely coupled.

The concept of coupling is how tightly integrated a function is to the project. Ideally you want functions to be loosely coupled. This way the can easily be ported to other projects. When you use global variables now you must take them with you when you want to port your code. 

 In addition, when variables are global, they can be seen and modified by any function anywhere anytime making it impossible to ensure the value of  the variable at any given time.
