 Overview of a Computer

 This is really a simplistic overview of a computer but then again this is a programming class and not a hardware class



Essentially, the PC consists of a CPU, memory (RAM), disk drives and miscellaneous IO devices.



All devices are connected to the CPU (Central Processing Unit) by a bus that allows data to travel in and out of the CPU.



When a program is launched its data goes from disk to memory and then into the CPU where it is executed.













Memory



In a windows machine memory is addressed as a series of bytes with each byte consisting of 8 bits. Memory can hold basically two things; program instructions, and data. The programs you create will write both types of data to memory. The program instructions are the C++ code that you write to tell the CPU how to execute and the data is the information you will use to work with your program. Data for instance could be a checking account balance if you were to write a checkbook program.

All memory in your computer is located at a specific address. C++ is a high level programming language so it abstracts the dirty detail of having to know what memory location that you are dealing with.







It is important right off the bat that you understand that everything you do with a program is going to use memory in one way or another. Understanding this will give you a head start when we talk about pointers

As stated earlier memory is stored in a byte wide fashion with each byte consisting of 8 bits. We should actually depict an area of memory like so:





Each box contains one bit each bit can be in either one of two states; on, or off. This is designated as 1 meaning on and 0 meaning off.

Creating a C++ Program

 

There are a few steps you need to go through in order to create and run a C++ program. First you create a text file and save it with the extension .cpp. The code that you write within this file needs to be written in ASCII text. This means that you cannot use a program like Microsoft Word because it does not save files in ASCII text. Word stores files with control characters that keep track of things like the bold text, etc. There are several text editors around, you can even use Notepad if you like but I don't suggest it.

In the old days of C we used to write all of our programs using a word processor called WordStar. It was cumbersome at best but was probably the best thing at the time for writing code. Fortunately for us, Microsoft has given us Visual Studio, which you will soon become well acquainted with.

Once you have created the text file, you need to write C++ code within the file that will accomplish some task. The code you write must follow the rules of the language. These rules are called the syntax of the language.

Preprocessing

 

This is the first step and happens prior to the compiler interpreting your code. The preprocessor will scan your code looking for preprocessor directives. The directives are easily recognizable because the start with #. The most common directive is the #include directive. It tells the preprocessor to replace the directive with the contents of a particular file. For instance #include <iostream> tells the preprocessor to include the file iostream. Most #include files contain definitions that the compiler needs to see to compile the source code. For instance iostream contains definitions for the built in functionality to put information to the screen and get it from the keyboard.

Compiling

 

Once you have created your .cpp file, it next needs to be ran through a tool called a compiler. The compiler goes through your source code file and checks to make sure that you have followed the syntax rules of the C++ language. If you have not typed something correctly, the compiler will spit out an error message that tells you something about the problem it has encountered. At first, these errors are going to be somewhat cryptic and not make much sense. You will find that with time you will be able to understand the error messages and be able to correct your errors.

The compiler spits out an object file with the extension .obj; this file contains machine code. When we talk about machine code, we mean the binary 1's and 0's that the processor understands as an instruction to perform a particular action. You can think of object files as an intermediary file that is close to being an executable but not quite. This is where the linker comes in.

Linking

 

In most software development, the size of the code starts getting huge. It would really be a burden on the development team if all of the source code had to be put into one file. First of all, the file would be huge. Think about a single file that contains 1 million lines of code. Another problem is that the development team could not easily distribute tasks. So it is the linkers job to put all of these files together so that the project can be ran as an executable.

One other point, and it is a little early to talk about but I think that it should be mentioned here. The C++ programming language comes equipped with functionality that you can use to accomplish programming tasks. This functionality as you will see is quite useful when creating a program. Imagine if you had to write every single piece of code!! It would really be a hassle for you to write simple things like putting text to the screen or getting data from the user. This is why C++ provides a library of functionality for you to use. This library is called the standard library and we will get into it in more detail as time goes on. What you should understand is that the definitions of functionality are provided through the use of #include files which are handled by the preprocessor. It is the linkers job to actually go out and get the code from the standard C++ library. 

For instance #include <iostream> has definitions for functionality to input data from the keyboard and output data to the screen. These definitions are used by the compiler to ensure that the programmer is using valid functionality. It is the linkers job to go out and get the code that corresponds to the definition.

You should note that it is possible to create a definition of functionality and use it but if the actual code does not exist then the linker is going to spit out error message "Unresolved External".

The reason the standard library is mentioned here is because it is also the linkers job to bring in the code that you are using from the standard library and place it in your program

Executable

 

If everything has been written correctly, the linker will spit out a file with the extension .exe. This is an executable file that can be ran by either double clicking on it or by typing its name at a DOS command prompt.

Where To Get Tools

 Before you can write your first program you need to install a compiler. These days we have what is called an IDE. This stands for Integrated Development Environement. The IDE really does all the work for you. Basically, you create a project, add files to it, and build. If you don't have any syntax errors then the IDE will spit out a program that runs.

By the way, syntax is the rules of the language, C++ in this case. You will be learning plenty of syntax as the course goes along. So be pepared to do a certain amount of memorizing.
