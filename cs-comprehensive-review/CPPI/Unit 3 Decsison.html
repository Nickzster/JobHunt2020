The bool Primitive Type

In a previous section you discovered that C++ has four primitive data types: whole numbers (integer), real numbers (floating point), characters and the bool types. The bool data type is the one data type that leads to a great deal of misunderstanding in programming. Because of this, and the fact that many things in programming boil down to truth, the bool type warrants it's own section. 

The bool type works the same way as any other primitive data type in C++. You can create both bool variables and bool literals. The main difference between a bool type and any other primitive data type is that the bool type can only hold one of two values, true or false. Because C++ is a case sensitive language and all of the keywords are lower case. You should beware that you do not  put any uppercase characters in the keywords true and false. 

You create a bool variable the same way you create any other variable type.  You start with the type (bool in this case), create a name, and then (optionally) initialize your variable.

Here is an example of two boolean variables definitions:

bool empty = true;
bool visible;

Naming bool variables:

When you choose names for variables you should try to create a name that has some meaning. Many times new programmers will create variables with the names like x and y. While these are short and easy to type they do not say anything about what is being stored in a variable. When you choose a name for a bool variable you may want to alter your thinking a little bit. Instead of choosing a name that describes what the variable holds you should choose a name the describes how the variable will be used. Try to name your variables so that when the value is true, your conditional expressions can be phrased "naturally":

if (visible) ...
if (finished) ...

We will see some examples of using the bool operator later on in this lesson. For now we need to learn a little more about boolean expressions.


The Relational Operators:

While bool literals and variables can at times be helpful, the best use of the bool type is to create Boolean expressions. A Boolean expression uses one of the relational operators in conjunction with two primitive type operands to form a conditional statement that yields a Boolean  (true / false) result. The C++ programming language has six relational operators to choose from:




The first four of these operators are called the inequality operators, while the last two are called the equality operators. Those operators that require two symbols [<=, >=, ==, and !=] must be typed with no space between the two symbols.

If you come from another programming language, here are some potential pitfalls you should look for:

  • The equality operator is ==, not = , which is the assignment operator. Be careful with this operator. It is the source for many errors. Remember, single = is for assignment, double == is used for checking equality.
  • The inequality operator is !=, not <>.
  • Make sure you use >= for greater-than or equal, and not =>, which is not a valid symbol.

 

Primitive Relations

 

What you should know from the reading so far in this section is that each relational operator requires two primitive operands. The result of the expression produces a Boolean (true/false) result.

You can only use comparable primitive types to form boolean expressions. Fortunately, most primitive types in C++ are comparable. This means that you normally don't have to be concerned about comparing mixed primitive types. Typically, they will work just like you would expect them to.

Let's take a look at a few examples:

Variables Comparisons

int b = 2;
int sh = 3300;
double d = 2.45;
float f = 2.45F;
char c = 'A';

 b > sh is false
c < b is false
f == d is false
f < d is true
c >= d is true








 

Floating Point Relations

Floating point numbers are really approximations so you should never compare them using the relational operators (!=, ==). As an example take the two numbers 2.45F and 2.45. On the surface it seems that these numbers should be the same. In reality, 2.45 is a double and 2.45F is a float which will make their bit patterns somewhat different. Aside from this there are certain numbers like .1 for instance that have a bit pattern that repeats to infinity. Actually, it repeats to the end of the size of the data type.  To get an understanding of how binary floating numbers are represented in your computer take a look at the lecture material on number types in the CSIS 118B Introduction to Assembly language.


Selection

It was stated earlier that many things in programming boil down to determining truth. You have seen that you can use a relational operator along with two numeric operands to form a boolean expression. The reason this is important is that we can use this to perform decisions in our code. A decision also called selection acts as a way to make your code branch in one direction or another. You might want to think of this branching like a highway divider in your code. 



With selection you can construct a boolean expression, test the result, and then branch your code down a path if the result of the test is true. Simply put, you can have specific code execute if the result of the boolean expression is true and skipped if it is false. As we get further into selection you will see that you can optionally have alternate code execute if the result of the expression is false. 


Here is an example of some pseudo-code that demonstrates how selection might work in your code:

If the number of items sold is less than 35000 then 
Calculate the bonus amount to be 3.5% 
otherwise 
Calculate the bonus amount to be 7.5%


Selection is also called branching  because any time you run the program you may take a different path through the code. C++ has five different kinds of branching or selection statements. In this section you will get to see the following:

  • The if statement
  • The if-else statement
  • Nested if and if-else- if statements
  • The switch statement
  • The conditional operator [?:]

Decisions With if

 

The if statement is the most simple way of doing selection in your code. Like a function (main for instance) the if statement is a block of code so it's body should have an open and closed brace. Like a lot of things in high level programming languages there are many shortcuts that you can use. We will see this here with the if statement where there are certain conditions under which the open and closed braces can be omitted. 

The first line of the if statement is called the header. It consists of the keyword if, followed by an open and closed parentheses. Within the parentheses you will use relational operators to construct a boolean expression to be tested. If the test expression yields a true value then the body of the if statement will be executed. If the result of the test is false the body of the if will be skipped.

The Logical Operators

As  you have been going through this material, you may have thought C++ seems to be a never ending stream of operators. In one sense you are correct. Like other programming languages C++ has many operators that need to be learned and understood. So, guess what? In this section we will be looking at more operators. 

The operators you will be seeing in this section are called the logical operators. One of the things all but one of these operators allows you to do is to make compound if statements. What do I mean by compound if statement? Well, let's look at the operators first then we will see how to make a compound if statement. 

C++ has three logical operators they are: 

  • The unary logical operator : NOT (!)
  • The binary AND operator (&&)
  • The binary OR operator (||)

The NOT Operator

The NOT operator is probably the best operator to start with because it is the one operator that you will NOT use to create compound if statements with. The NOT operator uses the exclamation point (!) which is placed in front of a bool operand and has the effect of flipping the value:



What is meant by flipping the value? Examine the following:

bool on = true;

bool off = !on;

If a value is NOT true it must be false and if it is NOT false it must be true. In the above example the variable on was set to true. On the next line of code the variable off was set to NOT, which has the effect of flipping the value from true to false.

Here is a few more examples:

15 == 7 + 5 + 3;

is true because 15 is equivalent to 7 + 5 + 3. Using the NOT operator you could write:

!(15 == 7 + 5 + 3);

which would be false.

AND OR Pitfalls

The AND and OR operators are easily confused. As you start using them you will at some time undoubtedly use the AND where you intend to use the OR and the OR where you intend to use the AND. There are two main logic errors that you need to concern yourself with, the Impossible condition, and the unavoidable condition. Let's take a look at each of them:

The Impossible Condition

The impossible condition happens when no matter what value the two operands are set the outcome of the statement can never be evaluated to true. 

Consider the following statement.

The unavoidable condition is when no matter what values are supplied for the two operands the result will always yield true. Previously, you saw that the impossible condition occurs when you use logical AND when you should have used logical OR.  The unavoidable condition is the opposite of this. It occurs when you use a logical OR where you should have used a logical AND.

Note that the relational operators all appear below the arithmetic operators, so you can write

a > 3 + 5

rather than

a > (3 + 5)

Note also that AND [&&] has higher precedence than OR [||]. This means that the expression:

10 < 15 || 5 > 8 && 3 > 5

which is true, instead of as

(10 < 15 || 5 > 8) && 3 > 5

which is false.

Multiple Choice

As you have seen, the use of the nested if and ladder-style if-else-if statements provide a pretty good mechanism for multi-way branching. C++ also has another mechanism, the switch statement which is better suited for selecting one of many alternatives. 

The switch statement works like the old telephone system that you may have seen in an old movie: The caller taps the receiver to get in touch with the operator, the caller then tells the operator who they would like to get in touch with, The operator then plugs the caller's cable into the correct place. 

While the switch statement is very efficient for doing multi-way branching it works somewhat differently than what you have seen before. With switch statements an integer constant is evaluated and the flow of the program branches to a matching case:

 

 

 

The switch statement has 5 different components: 

  1. The keyword switch, followed by parentheses which contain the integer expression to be evaluated. The switch statement is also a block of code so it is delineated by open and closed braces.
  2. Labeled case statements which include a constant integer expression followed by a colon. It should be noted that the switch statement can have any number of labeled case statements.
  3. The statement to be executed as a result of a matching case. You can have any number of statements needed. You can also include other blocks of code like decisions, looping structures, methods, and even other switch statements.
  4. A break statement which is needed to terminate each case. If the break is omitted the flow of  the program falls through to the next case. 
  5. An optional default statement that executes when there is not matching case. 

How the switch works

When the switch statement is encountered the integer expression is evaluated and the following steps are performed:

  • A matching case is scanned for and if found execution begins with the first statement following the colon of the matching case.
  • Each line of code within the case is executed until a break statement is encountered. If no break statements are found execution continues sequentially into the next case. This continues until a break statement is encountered or until there are no more statements within the body of the switch structure to execute. 
  • If no matching case exists execution will branch to the default statement where execution begins with the first statement following the colon. The default statement is optional. If it does not exist the code branches to the first line of code following the closing brace of the switch structure.

#include <iostream>

 

using namespace std;

 

int main()
{
int selection;

 

   cout << "Select a program to run " << endl; 
   cout << "1. Run Word " << endl;
   cout << "2. Run Excel " << endl;
   cout << "3. Run Powerpoint " << endl;
   cout << "4. Exit Program " << endl;
   cin >> selection;

 

   switch(selection)
   {
   case 1:
      cout << "Running word " << endl;
      break;
   case 2:
      cout << "Running Excel " << endl;
       break;
   case 3:
      cout << "Running Powerpoint " << endl;
      break;
   case 4:
      cout << "Good Bye " << endl;
      exit(0); 
   default:
      cout << "Invalid Entry " << endl;
   } 
   return 0;

}

If the user types in 2 for example, the switch will branch to case 2 and the code there will execute. In this case "Running Excel" gets printed to the screen.

Notice at case 4 the line that says exit(0). This is another function that is part of the standard library. It is used to terminate your program. You can use it anytime you want to stop your program.

What about the break statement

If you omit the break statement the code will fall through to the next brace. For example if  the break statement was omitted at case 2 then the output of the program would be first "Running Excel" printed to the console, followed by the code at case three which prints "Running Powerpoint" to the screen.
