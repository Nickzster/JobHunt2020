Introduction

I In this section we will be taking a hard look at the different data types you can use to create variables. A variable is like storage box that sits on a shelf in memory. You can use these storage boxes to temporarily hold data so that you can perform operations on it.


In this section we  will also learn how to use the mathematical operators to perform simple operations like:


  • add
  • subtract
  • multiply
  • divide

Before we do that though we should get some basic information out of the way:

Comments

A comment in any programming language is a place for you to create and leave notes to yourself or anyone else who may have to modify your code. You should know that comments are ignored by the compiler. This means that they are not part of the machine language. The C++ programming language supports two kinds of comments: single line, and multi-line. The single line comment  uses the double slash // :




The block comment (multi-line comment) uses a /* and */. The syntax for this comment was born out of the C programming language and looks like this:



You should use comments in your program liberally. It is a way to make your code much more readable and understandable. You will find as you progress with the C++ language that things can become quite complicated; comments will allow you to make your code self documenting and easier to follow.  


Case Sensitive

C++ is a case sensitive language; this means that 


main
Main
MAIN

are all considered to be different.

The fortunate thing is that all keywords in C++ are lower case!


Literals

A literal value is one that is literally stated. Here are some examples:

1
2.2
.3
'h'
"Hello"

The first three examples are pretty straight forward but the last two deserve a little explanation. When you want to literally state a single character you have to put single quotes around it. The last example is a string which is nothing more than a series of characters that is surrounded by double quotes. You have to put quotes around characters and strings so that the compiler does not confuse them for an identifier or variable. We haven't talked about variables yet but they are coming shortly.

ASCII Encoded

Anything that you type at the keyboard is a character. This includes numbers. What can be said is that there is a numeric equivalent for all characters. You may be thinking that the character 1 actually has a value of 1 but you would be wrong. The character 1 actually has a value of 49! This is because in C++ character use ASCII codes. ASCII was actually invented in 1960 but not published until 1963 it originally consisted of 127 characters that make up the characters that are now typed at the keyboard.

The first 32 characters are non printable characters called control characters. They were originally used to control the transmission lines on TWX (twix) systems which has its origins with the telegraph. Some of the characters are obsolete for instance ASCII 7 used to make a bell sound in your computer's speaker. In most cases the function to get data from the keyboard will do this for you. In other cases you are going to have to convert it your self. It is typical for you to need a place to store data when you write C++ programs. For instance, a program that calculates mortgage interest will need a place to store the principal value input by the user. This is where the variable comes in. A variable is a place in memory that will temporarily store data. 

 

Notice that the memory address is some huge hexadecimal number.  One of the nice things about high level languages is that you don't have to know what that memory address is. You simply give the memory address a name that is meaningful. This will make it easy for you to remember. In the above example, the variable is named myVar.

The next logical questions are: Ok, so how do I create one of these variables and how do I use it to store and retrieve information? The answers are really quite simple. To create a variable you state the data type you want to store and the name you want to assign to it.

This creates a variable that will hold an integer value and its name is myVar. The compiler will go out and find a chunk of memory and assign your name to it. You don't even need to worry about what address it is located at.

Variable Naming Rules

When you choose a variable name it should have some kind of meaning. We all have a tendency to give our variables names like x, y, or z.  These are not a very descriptive variable names if you plan on holding sales tax data for instance. 

Here are some rules for naming variables:

  • All variable names must begin with a letter, underscore, or dollar sign. You should refrain from using the dollar sign because Java uses this internally.
  • After the initial letter, variable names may also contain letters and the digits 0 to 9. No spaces or special characters are allowed.
  • Variable names can be of any length but you should try to keep them concise. Typing long variable names many times in a program is cumbersome. 
  • You cannot use a C++ keyword for a variable name.

Strings in C++

The  C++ language has primitive types int, char, float, double, bool, etc. This is somewhat limiting because there is no type built into the C++ language to hold text.  You have to remember that C++ was born out of the C programming language which was designed to be extremely low level for a high level language. To hold text in C you would use an array of characters. Because they were born out of C these arrays were called C_Strings.  By definition, a C_String is nothing more than an array of characters that is terminated with the null character ('\0'). Fortunately, when the Standard Template Library came about in C++ so did the string data type. The string data type is really a class (a complex type) which is something we will get into in the advanced class.  What you should know at this point is that complex types have functionality built into them.  Here, the string class is presented to provide you with an easy way of working with text.


To use a string, you must first include string into your project:

#include <string>

It is also part of the std namespace so you must still have:

using namespace std;

You declare a string the same way you would any other variable or object:

string str;

The string class has overloaded operators so you can assign data to it using the = operator:

str = "Hello World";

or you can initialize it at the same time you declare it:

string myString = "Hello World";

Of course if you don't want to include string, you can always use the scope resolution operator:

std::string s = "Hello World";

 Initialization (constructor)

  
A string object may defined without an initializing value, in which case its initial value is an empty string (zero length, no characters): string str1;

A string object may also be initialized with

  • a string expression:
    string str2 = str1; string str3 = str1 + str2; string str4 (str2); // Alternate form
  • a character string literal:
    string str4 = "Hello there"; string str5 ("Goodbye"); // Alternate form
  • a single character. Unfortunately, the expected methods don't work:
    string str6 = 'A'; // Incorrect string str7 ('A'); // Also incorrect

    Instead, we must use a special form with two values:

    string str7 (1,'A'); // Correct

    The two values are the desired length of the string and a character to fill the string with. In this case, we are asking for a string of length one, filled with the character A .

  • a substring of another string object:
    string str8 = "ABCDEFGHIJKL"; // Initialize str9 as "CDEFG" // Starts at character 2 ('C') // with a length of 5 // (or the rest of the string, if shorter)
    string str9 (str8,2,5);


 length, size
size_type length() const;
size_type size() const;

Both of these functions return the length (number of characters) of the string. The size_type return type is an unsigned integer type. (The type name usually must be scoped, as in string::size_type .)
string str = "Hello";
string::size_type len;
len = str.length(); // len == 5 len = str.size(); // len == 5
 


 c_str

 
const char* c_str() const;
For compatibility with "older" code, including some C++ library routines, it is sometimes necessary to convert a string object into a character array ("C-style string"). This function does the conversion. For example, you might open a file stream with a user-specified file name: string filename; cout << "Enter file name: "; cin >> filename; ofstream outfile (filename.c_str()); outfile << "Data" << endl;




insert

 
string& insert(size_type pos, const string& str);
Inserts a string into the current string, starting at the specified position.
string str11 = "abcdefghi";
string str12 = "0123";
str11.insert (3,str12);
cout << str11 << endl; // "abc0123defghi" str12.insert (1,"XYZ"); cout << str12 << endl; // "0XYZ123"

string& replace(size_type pos, size_type n, const string& str);
Delete a substring from the current string, and replace it with another string.
string str14 = "abcdefghi";

str14.replace (4,2,str15);
cout << str14 << endl; // "abcdXYZghi"






find
size_type find (const string& str, size_type pos);
Search for the first occurrence of the substring str in the current string, starting at position pos . If found, return the position of the first character. If not, return a special value (called string::npos ). The member function rfind does the same thing, but returns the position of the last occurrence of the specified string.
string str16 = "abcdefghi";
string str17 = "def";
string::size_type pos = str16.find (str17,0);
cout << pos << endl; // 3
pos = str16.find ("AB",0);
if (pos == string::npos) cout << "Not found" << endl;
