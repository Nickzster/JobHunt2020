What is a pointer

A pointer is a special type of variable. What is so special about a pointer you might ask? Well, a pointer is a variable that holds the address of another variable. Through the use of pointers, we can indirectly manipulate values at another memory location.

There are two symbols that you should know about right off the bat they are:

& - addressof operator (Yes it is the same symbol that is used for references)
* - de-reference operator (It may be easier to think of it as the contents of operator).

Still Confused?

 

Lets look at it this way, a pointer is just like any other variable. It sits in memory just like any other variable. The difference is it holds the address of another variable. To get at the value of the variable the pointer is referencing, you use the * operator:




In the above example you will notice that the variable x sits at address 0x500 and that the pointer sits at 0x900. The assignment statement says take the address of the variable x and store it at the location of the pointer ptr (0x900). So an address is stored at 0x900. It is the address of x in this example. The cout statement uses the * before the ptr which means get the contents of the variable it is pointing at. In this case it says get the value that is stored at memory location 0x500 which is 3.

Taking the above as an example what prints if I do this:



cout << ptr;   would print 0x500 because this is the address stored in the pointer variable.
cout << *ptr; would print out 3 because this is what the pointer is referring to (what is stored in x).
cout << &ptr; woud print out 0x900 because this is the address where the pointer sits in memory.

Of course you need to realize that these memory locations are for example only. The real addresses would be a big hex number.

Some Final Thoughts

Pointers were around long before references were invented. It almost seems like references were created to make the job of passing data by reference much easier. Also, some people think that pointers are dangerous and, they are. You could setup a pointer to point at just about any memory location and then modify that location. Of course on a Windows machine a lot of the memory is protected now. In the world of embedded systems where you don't have those restrictions, pointers come in handy. Lets say for instance that you need a way to write characters a display screen. All you would have to do is set a pointer equal to that location in video memory and start writing to it.

One of the knocks on references is that when you call the function you don't necessarily know that values are being passed by reference. Pointers get around this because you almost always have to pass the address of something.

Finally, I am going stress the use of pointer more than many other instructors probably would. This is because I feel it is one of the more powerful features of the language. You will probably find that you could write just about any program in C++ without the use of pointers but on occasion, the use of a pointer just might make your life easier.
