Introduction

So far we have only dealt with the primitive types of the C++ language. Primitive types are nothing more than the smallest types that a language can have. You know these as int, float, char, double, long, bool. Arrays are our first look into creating abstract data types. So what is an abstract data type you ask? Well it is nothing more than the putting together of atomic types to create a custom type; one that you define.

Definition:

An array is a contiguous group of homogeneous elements.

  • Contiguous meaning that they sit next to each other in memory one right after another in sequential fashion.
  • Homogeneous meaning that they all of the same type

Think of an array like a grocery list. In this case though you would put a number next to each item on the list which specifies the order in which the item occurs on the list.

 

You declare an array the same way you would any other variable. Give it a type, followed by a name, then you place an open and closed bracket with the number of elements that you wish to have. The above example declares an array of int's 5 elements long called myArray. You can add elements to an array by using the equal sign and the index of the location in memory you would like to place it.

With arrays the brackets serve double duty. At declaration time you put the number of elements that you want the array to hold. Any other time you place the number of the element that you wish to address. Above, we addressed element 1 and set it equal to 5. When accessing an array the number inside the brackets has a few different names. Some people will call it index and still others will call it subscript to name a couple. You will see me shorten the word subscript to sub in many of the videos on this sight. You would say that "myArray sub 1 is equal to 5 ". 

Arrays and Memory

Remember that everything in a computer program is located somewhere is memory All array elements start at 0 and you should think of element 0 as the base address of the array. When you put a number in the brackets, think of it as adding a number to the offset in memory from the base:

 

Because an int is four-bytes wide the addresses above increase by four. If the example was of a char the addresses would increase by one because a char is one-byte wide. Setting myArray[2] = 6 is like saying take the base address 0x500 and add eight to it (or 2 * 4-bytes) and place the value 6 at that memory location. Fortunately for us all of this arithmetic is figured out by the compiler. What you should note is that each of the elements sits next to each other in memory. If you know the address of the first element then you can easily deduce the address of any other element in the array. 

The above memory locations are not really accurate. They were picked simply for illustration purposes only. In reality the starting address would be some big hex number. Fortunately, you don't even need to think of the address or how many bytes apart each item is. You simply use an index which corresponds to the item on the list you want access to.

I have been preaching a lot about where things sit in memory but this is a real important point. We are going to be talking about pointers in the near future and understanding this concept will serve you well.

From the above you should be able to deduce that the subscript of an array is really just an offset in memory from the base address!

C Strings

C++ was born out of the C programming language. One of the beautiful things about C is that it is a very simple language. Other than assembly code, no other language is at a lower level than C. Because it was such a simple language, there was not a lot of frills added into it. If you have done any other high level languages like Java or Visual Basic then you are accustom to having a string as an atomic type of the language. C doesn't offer this and neither does C++. Later, you will find that someone has added a string class to the language but it was not originally there. This brings us to what is affectionately called C Strings. C strings are nothing more than an array of characters with a special marker added to the end. The marker is a binary 0 but most of the time you will see it written using the escape character '\0'

Vectors

The vector is our first look at the standard template library also referred to as the STL. The STL will be covered in more detail in the C++ level 2 class.  The classes that are part of the STL are also called container classes because they contain lists of information. Like all STL classes the vector acts like an array that will grow and shrink depending upon the amount of data put into it. What is nice about a vector is that you don't have to worry about all of the memory management that goes along with growing and shrinking an array.  The vector class handles it for you. You can think of a vector as a class that will hold any type of data and as much of it as needed. That is unless your run out of memory.

To use a vector you must include the vector header into your code:

#include <vector>

The declaration of a vector has the following syntax:

Using this syntax, a vector can easily be created:

vector <int> v;

So the above creates a vector called v that holds ints.

Within the brackets, you can plug in any type that you wish, even classes. Remember the Rectangle class?

vector <Rectangle> rect;

Like 

Arrays

Vectors can be used like an array. Meaning you can use array notation to access elements:

int x = v[0];

You need to be careful using this syntax when adding elements. If the vector does not have any size then you will get a bad error. Here is an example:

vector <int> v;   // Vector has no size 
v[1] = 30;    // Error Time

To fix this you can use the reserve function to create some size:

v.reserve(100);  // Create space for 100 ints
v[1] = 30;   // Works fine now that space has been created

You can also accomplish this with the constructor

vector <int> v(100);   // Create a vector with space for 100 ints
v[1] = 30;

If you don't want to set an initial capacity you can simply use the built in push_back function which creates space if there is none available:

vector <int> v;
v.push_back(32);

Determining the size of a vector:

 

  The vector class has a function called size() which returns the number of elements in the vector. This is a real advantage over arrays because arrays force you to pass around the size to each function that needs to manipulate it. You can now iterate through the vector using it's size() function:

for(unsigned int i = 0; i < v.size(); i++)
   cout << v[i] << endl;

The vector actually returns an unsigned int. This is because there should never need to be a negative amount of elements within it.

Vector Capacity


When the vector gets full and you add another element, the capacity increases. How much does it increase? Well, this is dependant on the implementation but many times it will double its capacity each time it gets full. Don't confuse capacity with size. The size is the number of elements in the vector and the capacity is how much it will hold. Most of the time the capacity is going to be much larger than the size.

It has been said that vectors are not very efficient. This is because of the way they handle memory (among other things). If memory issues are important, you can use the reserve function to specify how much more memory to allocate (instead of doubling it).

v.reserve(50); // Sets capacity to 50 elements
v.reserve(v.size() + 25); // Sets the capacity to 25 more than the current size

Iterators

 

All STL container classes contain iterators that are used as a means of iterating across the entire collection. The iterator is really like a type safe pointer and as a matter of fact is used like one. 

To create an iterator your use the scope resolution operator:

 vector<int>::iterator p;

This assumes that the collection you are to be iterating over holds type int.

To set the iterator to a starting point we use some of the built in functions that are designed for iterators:

vector <int> v;
vector <int>::iterator p;

p = v.begin();

You can also set it to the end:

p = v.end();

These two functions are useful in for loops to iterate across the collection:

for(p = v.begin(); p < v.end(); p++)

notice that the iterator is used just like a pointer! If you wanted to output what the iterator is pointing at you would treat it like a pointer also:

cout << *p << endl;
