Introduction

We haven't had a chance to cover inheritance yet so this may be a good time to speak a little about it. Inheritance allows you to derive one class from another. Typically you have general class, called the base class that has the basic functionality in it. If you need to create a specialized version of the base class you can simply derive the specialized class from the base. What this allows you to do is use the functionality from the general class and add the specialized functionality to it. As you can see inheritance promotes code reuse.

In C++ reading and writing to files is made quite easy through the use of inheritance. Because all input and output classes are derived from the same parent they all have many things in common. A file stream then is a specialize version of the stream class. Just as console input and output are. You will find then that writing to a file is very similar to writing to the console window and reading from a file is very similar to reading from the keyboard.

To understand the base / derived class relationship it may be best to see an inheritance chart.

You have been using streams for a quite a while now primarily in the form of cin and cout. In this section we are going to take the concept even further by investigating ifstream class for reading from files and the ofstream class for writing to files.

Sequential Access Files

Sequential access file IO is pretty much self explanatory. You open file and read sequentially from the top of the file to the bottom. The process of reading and writing to a sequential access file is not that much different than reading or writing from the console. There are three things you need to know

  1. Open the file - Reading uses an instance of ifstream and writing uses an instance of the ofstream class
  2. Read or Write to the file - Use some of the stream functionality to get or write data to and from the file
  3. Close the file - It is important to close the file as soon as you are finished reading or writing. Files left open can become corrupt.

In order to use file IO classes you need to first include frstream. The fstream header has defintions for ifstream (input file stream) and ofstream (output file stream). If you watn to read from a file you need to create an instance of the ifstream class and if you want to write to a file you want to create an instance of the ofstream class.

1. Opening a File

There are two ways to open a file for reading or writing. Both methods involve the create of either an ifstream object or an ofstream object. You can pass the name of the file to the constructor or you can use the open function that is part of the object

ifstream infile("fileName_To_Open");  // Use overloaded constructor to open file

ifstream anotherFile;
anotherFile.open("FileName_To_Open);  // Call the file open method after the creation of the file object

Files actually have two names. The first is the actual filename also called the physical file name. In the above example the physical file name is called "fileName-To-Open". The other name for the file is the logical filename. The logical filename is the name of the object created. In the above example the logical filename is infile. Once the file is open you will almost always use the logical filename when reading and writing.

Checking For File Open

When you attempt to open a file you should always check to make sure the file opened properly. The stream classes make this easy by overloading the ! operator. It returns a non zero value if the file could not be opened. This is not something that happens a lot but it is possible that the file does not exist, is opened by another application, and things of this nature.

There are two ways to check for file open failure. The first is to use the iso_open function:

 ifstream infile;
infile.open("somefilename"); 
if(! infile.is_open())
{
       // handle error here
}

The other way is to simply use the ! (Not) operator in a decision with the object:

ifstream infile;
infile.open("somefilename"); 
if(! infile)
{
       // handle error here
}

File Paths

The folder hierarchy on your computer is setup as a tree structure. A file path then is a way to specify where on the tree the file is located. There are basically two types of paths:

  1. Absolute Paths
  2. Relative Paths

An absolute path is simply the complete path to the file. This starts from the root of the tree. An example of an absolute path would be:

C:\Files\Documents\main.cpp

At the root of the tree you have the drive letter (C: in this case). After the drive letter comes a backslash which indicates the root of the tree. Each folder then is separated by a \ working your way down the tree to the file.

Using an absolution path is really not a good idea because if you move the app to another computer the odds of the drive tree structure being the same is extremely low. When you have to use a path to specify a file location you should try to use a relative path whenever you can

A relative path is the location of the file relative to the application. If you move the app you simply need to keep that path of the file relative to the application the same. This is a pretty simple thing to do.

To start with we need to discuss a couple things that will make working with relative paths much easier. The first is . (period) the . means starting from the current folder as in:

.\files\main.cpp

The above says from the current folder go to a folder named files which contains the file main.cpp

The second is the double period .. This means go up a folder from the current folder:

..\..\files\main.cpp

The above says go up two folders then to the folder called files and get main.cpp within the folder named files

The \ Escape Character

In many of the popular high level programming languages today the backslash (\) character is an escape character. This tells the compiler to treat the next character as special /n for instance puts a newline in a string. Because of this anytime you want to specify a path you need to use the double backslash //

char filePath = "..\\..\\files\\main.cpp"

Visual Studio Paths

If you do not specify a path and simply put the name of the file you want to operate on the path defaults to the current folder. In other words the folder where the application is running from

int main()
{
	//No path defaults to application folder
	ofstream outfile("test.txt");

	return 0;
}

Visual Studio organizes it's project by creating a project folder inside your Documents folder. It is typically Documents\Visual Studio 2015\Projects\Project Name. If you are using a different version of Visual Studio then the path changes to the name of the version you are using. When you do not specify a path in Visual Studio the file will be written to the same folder where the source code files like main.cpp have been created.

2. Reading and Writing

Writing

The beauty of the C++ method of handling files rests in the simplicity of the actual functions used in basic input and output operations. Because C++ supports overloading operators, it is possible to use << and >> in front of the instance of the class as if it were cout or cin. In fact, file streams can be used exactly the same as cout and cin after they are opened. 

#include 
#include 

using namespace std;

int main()
{
    //Create an instance of ofstream and open example.txt 
    ofstream outfile("example.txt");

    //Make sure the file was opened
    if(!outfile)
    {
        cout << "Error opening file " << endl;
        exit(0);
    }

    //output text to the file
    outfile << "This text will no be inside of example.txt" << endl;
  
    return 0;
}

Reading

Reading from a sequential access file is no different than reading input from the keyboard. You can use the stream extraction operator (>>) with the file object to read data from the file.

int main()
{	
	ifstream infile("test.txt");
	if (!infile)
	{
		cout << "Error opening file for reading" << endl;
		exit(-1);
	}
	char input[100];
	infile >> input;
	cout << input << endl;

	return 0;
}

As with cin, using the stream extraction operator will only get you the first word in the file because the stream extraction operator terminates reading at a white space. If you want to read all of the words using this method you would have to use a loop:

int main()
{	
	ifstream infile("test.txt");
	if (!infile)
	{
		cout << "Error opening file for reading" << endl;
		exit(-1);
	}
	char input[100];
	while(infile >> input)
		cout << input << endl;

        infile.close();
	return 0;
}

Of course if you want to read an entire line from the file you could always use the getline function that is part of the ifstream object

char input[100];
infile.getline(input, 100);
cout << input << endl;

To read every line you could use a loop along with the getlline function.

char input[100];
while(infile.getline(input, 100))
	cout << input << endl;

The file funcitonality was built into C++ before the string class was added to the language. This is why the above examples use CStrings and not the string class. If you want to use the string class to get a line of text from the file then you need to use the getline function:

string input;
while(getline(infile, input))
	cout << input << endl;

infile.close();

Random Access Files

Random access file allows you to write records to a file sort of in the way databases did it before the advent of the relational database. The problem with random access files is that the records need to be a fixed length so that you know where one record ends and the other record begins. You can create a record out of a structure but again you must make sure that it is of a fixed length. This means no vectors or classes can be included as part of your structure. In other words, you must use primitive types. 

seekp and seekg

When you open a file for reading or writing a file pointer is created that points to the current position in the file. If you have done any file IO in C you probably already understand this. When you read a byte from a file the file pointer moves to the next byte to be read. What you are reading then is the data located at the place the file pointer references. This whole notion of the file pointer is encapsulated in the stream classes and in most cases you don't need to be aware of it. If you need to move around the file in a byte-wise fashion you can use the seekp and seekg functions. Both of these move the file pointer around.

seekp and seekg basically work the same way except that seekp is part of the ostream class (used for writing) and seekg is part of the istream class (used for reading). Both of these functions allow you to move the file pointer to any position within the file. seekp and seekg are overloaded functions with the following formats:

  • ostream &seekp(streampos pos);
  • ostream &seekp(streampos off, ios_base::seekdir dir);

The first function moves the file pointer to a specific offset from where it is currently located. The second function allows you to specify a starting location to offset from:

Type of Seek Argument to seekp
Seek relative to beginning of file ios_base::beg
Seek relative to the end of file
 ios_base::end
Seek relative to current position
 ios_base::cur

The funtion seekg works the exact same way except you use it when you are reading files with an ifstream object

tellp and tellg

Like seekp and seekg, tellp is part of the ostream class and tellg is part of the istream class. Both return the current position in bytes from the beginning of the file to the current position of the file pointer. The prototype for the function is:

streampos tellg();

If you are wondering what streampos is you can find it in the iostream header. It is defined as follows:

typedef long streampos;

As you can see, streampos has been defined as a long data type.

To understand how seekp, seekg,tellp, tellg work let's look at an example that will determine the size of a file in bytes. The idea here is that the seek pointer will be moved from the beginning of the file to the end. Then tellg will be used to return the position of the pointer which happens to be in bytes:



#include
#include 
#include 

using namespace std;
bool writeFile(string fName);
int fileSize(string fName);

int main()
{

   writeFile("example.txt");
   int size = fileSize("example.txt");
   cout << "The size of the file in bytes is " << size << endl;
   return 0;
}

int fileSize(string fName)
{
   ifstream infile(fName);
   if(!infile)
      return 0;

   infile.seekg(0, ios_base::end);
   int bytes = infile.tellg();
   infile.close();
  
   return bytes;
}
bool writeFile(string fName)
{
   ofstream outfile(fName);
   if(!outfile)
      return false;

   outfile << "Now is the time for all good men to come" << endl;
   outfile << "To the aid of their country" << endl;
   outfile << "The quick brown fox jumped " << endl;
   outfile << "over the lazy old dog" << endl;
   outfile.close();
}

Writing Records

In order to write a record to a file it needs to be of fixed length. This way we can use seekp or seekg to move to the file pointer to the appropriate number of bytes for the beginning of each record. This allows us to access records randomly simply by computing the byte offset to the starting point of the record we want to read or write from.

Here is an example of a fixed length record that is 100 bytes in size:

struct guest
{
   char firstName[50];
   char lastName[50];
};

Writing fixed length records in many ways is similar to writing sequential access files.

#include
#include 

using namespace std;

struct Guest
{
   char firsName[50];
   char lastName[50];
};

int main()
{
Guest g;

   strcpy(g.firsName,"Glenn");
   strcpy(g.lastName,"Stevenson");
   ofstream outData("guests.dat", ios::binary);

   if(!outData)
   {
      cout << "Unable to open the file for writing " << endl;
      exit(0);
   }
   outData.write(reinterpret_cast (&g), sizeof(Guest));

}

Notice that the file is opened for binary writing. Binary files are more efficient than text files and are perfect for dealing with records. In the above example, only c_strings are being written but the structure could very well contain numbers. Writing everything in binary takes care of any conversions from numbers to strings.

Like arrays, all members of structures sit next to each other in memory. We want to write them to the file just like they were an array that is why the reinterpret cast is used so that the data can be written like one big chunk of contiguous byte wise piece of memory.

Also note that the sizeof operator is being used. This will tell the write function how many bytes we want written.

Reading a Record

Records are read in the same manner as they were written, like one big chunk of byte wise data. Here is a some code that shows how to read a record:

#include
#include 

using namespace std; 

struct Guest
{
char firsName[50];
char lastName[50];
};

int main()
{
Guest g;

   ifstream inData("guests.dat");
   if(!inData)
   {
      cout << "Unable to open the file for reading " << endl;
      exit(0);
   }
  
   inData.read(reinterpret_cast(&g), sizeof(Guest));
   cout << g.firsName << " " << g.lastName << endl;
}

Reading a Specific Record

The above functionality can be modified to read a particular record. All that needs to be done is to add a seekp statement to move the file pointer to the beginning of the particular record to read. In this example, the record number is passed to the function and used along with seekp to compute the location of the beginning of the record specified: 

void showRecord(int recNumber)
{
Guest g;

   ifstream inData("guests.dat");
   if(!inData)
   {
      cout << "Unable to open the file for reading " << endl; 
      exit(0);
   }
   inData.seekg(recNumber * sizeof(Guest)); 

   inData.read(reinterpret_cast(&g), sizeof(Guest)); 
   cout << g.firsName << " " << g.lastName << endl;  

}

Introduction

Namespaces allow you to group a set of global classes, objects and/or functions under a name. To say it another way, they serve to split the global scope in sub-scopes known as namespaces which use the following form: 

namespace identifier
{
    namespace-body 
}

Where identifier is any valid identifier and namespace-body is the set of classes, objects and functions that are included within the namespace . For example:

namespace general
{ 
     int a, b;
}

In this example a and b are normal variables integrated within the general namespace. In order to access these variables from outside the namespace you have to use the scope operator ::.

general::a
general::b

The functionality of namespaces is specially useful in case there is a possibility that a global object or function has the same name as another one, causing a redefinition error. For example:

#include  
using namespace std;

namespace first
{
   int var = 5;
}

namespace second
{
   double var = 3.1416;
} 


int main ()
{
   cout << first::var << endl;
   cout << second::var << endl;
   return 0;
}

using namespace

The using directive followed by namespace serves to associate the present nesting level with a certain namespace so that the objects and functions of that namespace can be accessible directly as if they were defined in the global scope. Its utilization follows this prototype: 

using namespace identifier ;

Lets take the above example and modify it so to use the using keyword

#include <iostream>
using namespace std;
namespace first
{
   int var = 5;
}
namespace second
{
   double var = 3.1416;
}
int main ()
{
   using namespace second;
   cout << var << endl;
   cout << (var*2) << endl;
   return 0;
}

In the above example we have been able to use var without having to precede it with the scope resolution operator.

The clause using namespace has validity only in the block in which it is declared (A block is a group of statements within brackets {} ) or in all code if it is used in the global scope. For example, if we had intended to first use the code of a particular namespace and then the code of another namespace within a function we could break sections into blocks:

#include 
using namespace std;

namespace first
{
   int var = 5;
}

namespace second
{
   double var = 3.1416;
}

int main ()
{
   {
      using namespace first;
      cout << var << endl;
   }

   {
      using namespace second;
      cout << var << endl;
   }
   return 0;
}

One common mistake people make with namespaces is that they believe everything needs to be in one file in order to wrap the code in the namespace. This is not the case. YOu simply need to wrap your code inside a namespace declaration in each file. The following shows Rectangle.cpp and Rectangle.h with a namespace

namespace glenn  //Namespace declaration
{
	class Rectangle
	{
	private:
		int length;
		int width;

	public:
		
		Rectangle(int length, int width);
		Rectangle();
		~Rectangle();
		int area();

		void setValues(int length, int width);  //Mutator
		int getLength();   //Accessor
		int getWidth();    //Accessor

		Rectangle add(const Rectangle &r);
		Rectangle operator +(const Rectangle &r);		
	};
}// end namespace block 
namespace glenn  //namespace declaration
{
	Rectangle::Rectangle()
	{
		Rectangle(1, 1);
	}
	Rectangle::Rectangle(int length, int width)
	{
		this->setValues(length, width);
	}	
	Rectangle::~Rectangle()
	{		
	}
	void Rectangle::setValues(int length, int width)
	{
		this->length = length;
		this->width = width;
	}
	int Rectangle::getLength()
	{
		return this->length;
	}
	int Rectangle::getWidth()
	{
		return this->width;
	}
	Rectangle Rectangle::add(const Rectangle &r)
	{
		Rectangle tmp(this->getLength() + this->getLength(),
			this->getWidth() + this->getWidth());
		return tmp;
	}
	Rectangle Rectangle::operator +(const Rectangle &r)
	{
		return this->add(r);
	}
	int Rectangle::area()
	{
		return this->getLength() * this->getWidth();
	}
} // End namespace block

NOTE:

While it is an easy thing to do,  putting the using namespace declaration inside a header file should be avoided. When you do this you are now forcing every file that includes your header to that particular namespace. If you want to use a using statement you should do it this way:

using std::string;
using std::endl;
using glenn::rectangle

Alias Definition

You can define an alias for a namespace if you wish. An alias is nothing more than allowing you to specify a namespace by a different name. The syntax for this is

namespace new_name = current_name;

The following example creates an alias called glenn for the std namespace

 #include 

//Give std an alias
namespace glenn = std;

int main ()
{
   glenn::cout << "Hello from Glenn" << glenn::endl;
   return 0;
}

Namespace std

One of the best examples that we can find about namespaces is the standard C++ library itself. As defined in the ANSI C++ standard, all the classes, objects and functions of the standard C++ library are defined within namespace std .

Namespace std is nothing new, you have been using it since the beginning of C++ Level 1. But you should know that the use of std is a recent addition to the language. As a matter of fact, it is almost as recent as the ANSI standard itself (1997) and many older compilers do not comply with this rule.

Almost all compilers, even those complying with ANSI standard, allow the use of the traditional header files (like iostream.h , stdlib.h , etc), and you can use them just as you would namespaces. The ANSI standard has completely redesigned these libraries taking advantage of the templates feature and following the rule to declare all the functions and variables under the namespace std .

The standard has specified new names for these "header" files. Basically using the same name for C++ specific files but without the ending .h . For example, iostream.h becomes iostream .

If we use the ANSI-C++ compliant include files we have to bear in mind that all the functions, classes and objects will be declared under the std namespace. For example: 

// ANSI-C++ compliant hello world
#include <iostream>

int main ()
{
   std::cout << "Hello world in ANSI-C++\n";
   return 0;
}

Or

// ANSI-C++ compliant hello world (II)
#include <iostream>
using namespace std;

int main ()
{
   cout << "Hello world in ANSI-C++\n";
   return 0;
}
