Introduction

In level 1 you learned about how you could overload function to allow them to perform same operations on different data types. The C++ language also allows you to overload most of the operators so that they can be made to operate on data types that you create.

Imagine for instance that you had created a Fraction class and you want to be able to add two Fractions together. You would probably then create an add function:

Fraction F1, F2, F3;
Fraction F3 = F1.add(F2);

This certainly works but using an operator overload would make this code much more readable:

Fraction F1, F2, F3;
Fraction F3 = F1 + F2;

As you can see this makes the code more intuitive and readable.

By default the + operator will not work on the Fraction class. Just as it will not work on any complex type that you create. In order to make this work you need to overload the operator so that it works with your data type and performs the functionality that you specify.

Operator Overloading Syntax

Operator overloading has the following syntax:

For many people operator overloading can be a confusing topic. It is actually an easy thing to do. When starting out the easiest thing t to do is to simply write the function the way you normally would. Once you have done this place the keyword operator after the scope resolution operator and replace the function name with the operator you want to overload.  

Let's start with a simple example of overloading the + operator in the Rectangle class. First we want to create the function called add that returns a Rectangle and takes a Rectangle as an argument:

Rectangle Rectangle::add(const Rectangle &r)
{
}

To change this so that we overload the + operator we need to add the keyword operator and then replace the name of the function with the actual operator we are overloading (+ in this example): 

Rectangle Rectangle::operator +(const Rectangle &r)
{
}

About Passing Parameters

 

In the above example a Rectangle class is being passed as one of the arguments. When you pass complex types (classes, etc.) it is best to pass them by reference. The reason for this is that by default C++ passes everything by value. This means a copy of the object gets put on the stack. You really have to ask yourself what is more efficient passing one 32-bit address on the stack or an entire object?

Also note that we do not want the mutator method to be able to change the values of the Rectangle class. To remedy this we pass the Rectangle class by reference as a const parameter. Let's look at the Rectangle class again:

If you were to compile this what you might find is that the add function might show an error that says: "Object has type qualifiers that are not compatible with the member function".

This is a little cryptic but what it means is that you are trying use an instance of the Rectangle as const but the function you are calling is not a const function. This can be easily fixed by making the functions const as well:

int Rectangle::getLength() const
{
	return this->length;
}
int Rectangle::getWidth() const
{
	return this->width;
}

This const declaration needs to be added to definition in the header file also:

int getLength() const;
int getWidth() const;

The add function and the overloaded operator may seem a little bit strange to you. Remember, when you overload an operator you want it to act like it would with a primitive type. Take for instance the following example

int x = 3, y = 2;
int z = x + y

Here the value that is in x is added to the value that is in y and the result is stored in z. Note that neither x or y has changed. The same holds true for the operator overload. The value of one Rectangle is added to another and the result is assigned. Examine the code one more time 

Rectangle Rectangle::add(const Rectangle &r)
{
	Rectangle tmp(this->getLength() + this->getLength(), 
		this->getWidth() + this->getWidth());
	return tmp;
}
Rectangle Rectangle::operator +(const Rectangle &r)
{
	return this->add(r);
}

In the example a tmp Rectangle is created to store the sum of the current object with the Rectangle parameter. This would be the sum of the length and the sum of the width. The result is stored in tmp then returned. When a call is made:

Rectangle r1(2,3), r2(3,4), r3;
r3 = r1 + r2;

r2 would be the parameter to r1 and the sum of the two would then be returned from r1 and stored in r3. What is being stored is the tmp Rectangle that was created to hold the sum of the two.

Overloading the assignment operator

Overloading the = operator is not much different than overloading any other operator. As with all operators you want to make sure it operates the way the assignment operator should. There is one caveat you need to be aware of and we will look at it. First let's look at code to overload the = operator. 

Rectangle& Rectangle::operator = (const Rectangle &r)
{
   return this->equal(r);
}
Rectangle &Rectangle::equal(const Rectangle &r)
{
     this->setValue(r.getLength(), r.getWidth());
    return *this;
}

As it should, the = operator calls the equals method which simply sets the length and width of the current object. The caveat is that *this is being returned. The return type says that we are returning a reference to a Rectangle. Since we want to return the current object as a reference the current object pointer needs to be dereferenced (return *this). The reason the current object needs to be returned with an assignment operator is because of the problem with chaining

Chaining

It may be best to describe chaining with an example:

Rectangle r1, r2, r3(2,4);
r1 = r2 = r3;

Assignment in C++ goes from right left so here r3 is assigned to r2 and r2 is then assigned to r1. In the expression r2 = r3 causes r3 to be assigned to r2 then the value of r3 is returned. The return value is then stored in r1. Without returning the dreferenced this pointer the chain gets broken. This means that r2 will never get assigned to r1. Think of it this way: r3 gets assigned to r2 through the overloaded operator the new value assigned to r2 would then need to be returned in order for r2 to be assigned to r1.

A Couple of More Examples

Let's start wit the += operator. The += operator increases a value by a specified amount. In the case of the Rectangle we would want to add to the length and the width of the current object:

Rectangle &Rectangle::operator+=(const Rectangle &r)
{
   this->length+=r.getLength();
   this->width+= r.getWidth();
  return *this;
}

Notice that the += operator also returns the current object by dereferencing this. The reason this is done is to allow code like the following to work:

Rectangle r1(2, 3), r2(1, 1), r3;
r3 = r1+= r2;

Overload the equality operator

The == operator produced a Boolean result so it is natural for an overloaded operator to return the same:

bool Rectangle::operator==(const Rectangle &r)
{
   return (this->getLength() == r.getLength()) && (this->getWidth() == this->getWidth()); 
}

This will allow you to compare Rectangle objects

if(r1 == r2)

Overloading an Overloaded Operator

This may seem strange to you but there may be times when you want an operator to work on different data types. Take the Rectangle for instance. The overloaded == takes a class as an argument. The function then checks to see if the length and the width are the same in both classes. If they are the function returns true. Otherwise the function returns false. What if you wanted the ability to also check to see if the areas were the same. You could simply overload the == operator to take an int also. Remember the rules for operator overloading are the same as for normal function overloading. This means the parameters have to be different in some way. Here is an example of overloading the overloaded == operator:

bool Rectangle::operator == (int theArea)
{
      return this->area() == theArea;
}

Introduction

Friend functions go against the fundamental principles of Object Oriented programming because they allow a function that is outside of a class to have access of the private data of the class simply by declaring it a friend of the class. You will find that most Object Oriented philosophy is against friend functions. I am one that says that while it does violate object oriented principles in certain circumstances it can be OK.  

One very good reason to use friend functions is to add stream functionality to your class. Currently with the Rectangle class you can not use cin or cout to get input and output for your class:

Rectangle rect
cin >> rect;
cout << rect << endl;

The problem is that cin and cout have no idea how to deal with a Rectangle. This is the same case for any complex type created after cin and cout were added to the language. Fortunately C++ allows you relate cin and cout to your class by making the stream extraction (>>) and stream insertion (<< )  operators friends of your class. In addition to making the operators friends of your class you are also overloading the stream insertion  and stream extraction  operators.

Overloading The Stream Operators

Overloading the stream extraction and stream insertion operators is a simple thing to do. Since they are not members of a class it a little different but not much:

istream &operator>>(istream &input, Rectangle &rect)
{
   cout << "Enter the length of the Rectangle" << endl;
   input >> rect.length;
   cout << "Enter the width of the Rectangle" << endl;
   input >> rect.width;
   return input;
}
ostream &operator<<(ostream &output, Rectangle &rect)
{
    output << "The area of the Rectangle is " << rect.area() << endl;
    return output;
}

Before you run off and try this I need to tell you that it will not work the way that it is. This is because the overloaded operator is trying to access the private members of the Rectangle class. Remember, the operator would have to be part of the class in order for it to have access to the private data. Before we discuss fixing this I think a little discussion on what the code is attempting to do is warranted.

notice the stream extraction operator has two parameters. This graphic shows how you should think of the overload:

This is very much like passing cin to the overloaded >> operator function. In the example of using an instance of the Rectangle, input becomes an alias for cin. Of course this is a variable name and could be called anything but input seemed appropriate enough. What should deduced here is that cin is an instance of the istream class. It should also be deduced that that cout is an instance of the ostream class.

Friends

It may seem logical that the way to fix the problem above is to simply add the stream extraction operator to the class. This can't be done because of the way the function for the operator is defined. It needs values to the left and right of it. Essentially two parameters, the input stream and the data to operate on.

The solution to fixing this is to declare the overloaded operator to be a friend of the class. This is done simply by putting the keyword friend in front of the prototype inside the class. When you declare a function to be a friend of the class it now has direct access to private variables. Almost as if it were a part of the class.

Introduction

Lambda expressions have become very popular in recent years. Most of the popular high level languages like Java have support for Lambdas. This concept was added to the C++ language as of version 11 of the C++ standard. A Lambda expression in my opinion is like a local inline function. When the compiler sees an inline function it takes the machine language for the function and places it everywhere the inline function is called. This means that inline functions are very efficient because there is no overhead incurred for calling the function like there is with a regular function. A normal function has to store context, push values onto the stack, and branch to the location in memory where the function being called is located. This is very inefficient. Of course inline functions will make your compiled code larger because the code is being substituted at each location the inline function is being called

Lambda expression work much the same way. They are declared locally which reduces the overhead. If you are going to use lambda expressions it is probably a good idea to make them small functions that can be created in a few lines of code

Lambda Basics

To start with it is probably a good idea to see the syntax of Lambda expression so that each item can be understood

 

 

Number Name   Description 
 1 Capture Clause  All Lambda expressions start with the capture clause. The values for the capture clause are covered later.
 2 Parameters The optional parameters to the Lambda are provided within the parentheses. If no parameters are specified the parentheses are left empty
 3 Arrow Operator The arrow operator indicate the beginning of the return type. If the expression does not return a value the arrow operator must be omitted.
 4 Return Type The return type is like the return data type for any function. If the expression does not return a value this should be left blank
 5 Body This is the body of the Lambda expression. This is equivalent to a function body. 
 6 Parameter Value  If the expression takes parameters you specify the value(s) for the parameters here. 

At this point it is probably best to look at an example. In most programming languages we start be looking at a hello world application. In this case we are going to look at a hello world Lambda expression

int main()
{
    [](){cout << "Hello World" << endl; }();
    return 0;
}

  1. All Lambda Expression start with the capture clause. In this simple example the capture clause is left empty
  2. There are no parameters for the expression so the parentheses are left empty.
  3. There is no return type for the expression so the arrow operator is left out.
  4. Since there is no return type the value after the arrow is left out
  5. The body of the expression simply outputs hello world to the console window
  6. There are no parameters so no value is supplied for parameters. The parentheses are left empty

Supplying Parameters

Supplying parameters to the expression is a simple thing to do. The following shows a Lambda expression that will square and return a value passed to it. 


int main()
{
	int valToSquare = 4;
	
	int result = [](int num) -> int {return num * num; }(valToSquare);
	cout << result << endl;
	 
	return 0;
}

What has changed here is that a parameter is added inside the parentheses, the arrow operator is inserted followed by the data type to be returned, and the value to be operated on is supplied in the last set of parentheses

Calling a Lambda Multiple Times

Using the above syntax will make it difficult if you need to use the expression multiple times. On the surface it looks as though you would have to duplicate the code each time that you wanted to use it. This is where the auto keyword comes in. The auto keyword allows you to create a pointer to the Lambda Expression

int main()
{		
	auto sq = [](int num) -> int {return num * num; };

	cout << sq(4) << endl;	
	cout << sq(6) << endl;
	 
	return 0;
}

Here a pointer called sq is created so that the Lambda can be called many times. Notice the last set of parentheses are left off of the expression. This allows you to supply the value through the pointer like you would with a function call. 

Capture Lists

A capture list allows you to specify a means by which you can access variables that are a part of the surrounding scope. Surrounding scope means the braces that surround the Lambda expression. In the examples above the surrounding scope is main. Some of you are going to like this because it allows you to create a function where you don't have to pass parameters. To use a capture list you simply put the method you want to capture the variables by in the opening brackets []

As with functions there are two ways you can capture surrounding scope variables

  1. Capture by value [=]
  2. Capture by reference [&]

Capture by Value

Using capture by by value works the exact same way passing parameters does. A copy of what is in the variable gets passed to the Lambda expression. Since it is a copy the Lambda expression cannot change it's value to the surrounding scope.

Capture by Reference

Using capture by reference allows you to pass the address of a variable to the Lambda expression. As you have probably guessed, this works the exact same way that passing a parameter by reference does. If you change the variables value within the Lambda expression it is changed in the surrounding scope

It may be best to see how this works. The following example uses a Lambda expression to fill a vector. This is done by using capture by reference. The second Lambda expression simply prints out the vector. This time using capture by value. One thing that I should point out and that is the default method used is capture by value. Since it is the default method you may get an error by placing the = inside the bracket 


int main()
{		
	vector  v;

	//Capture by reference
	[&v](int size) { for (int i = 0; i < size; i++) v.push_back(rand() % 100); }(25);

	//Capture by value = is default so not used. 
	[v]() {for (int i = 0; i < v.size(); i++) cout << v[i] << endl; }();
	 
	return 0;
}

Of course if you wanted to be able to use the Lambda many times and fill the vector with different sizes you may opt to create a pointer:


int main()
{		
	vector  v;

	//Capture by reference
	 auto fill = [&v](int size) { for (int i = 0; i < size; i++) v.push_back(rand() % 100); };

	 fill(25);

	//Capture by value = is default so not used. 
	[v]() {for (int i = 0; i < v.size(); i++) cout << v[i] << endl; }();
	 
	return 0;
}

There is one last capture list type and that is this which allows you to capture variables from the surrounding class.
