Introduction

Before looking at dynamic memory allocation, we should probably have a review of pointers. Pointers tend to be a difficult topic for most people but without a good understanding there is no way we are going to be able to understand how to dynamically create variables.

A pointer is a special type of variable. What is so special about a pointer you might ask? Well, a pointer is a variable that holds the address of another variable. Through the use of pointers, we can indirectly manipulate values at another memory location.

There are two symbols that you should know about right off the bat they are:

  • & - addressof operator (Yes it is the same symbol that is used for references)
  • * - de-reference operator (It may be easier to think of it as the contents of operator).

Review

When you create variables like the following:

double myDouble;
char myChar = 'A';
int myInt = 2468

The compiler allocates enough memory to hold the specified type, creates an association between the name of the variable and the allocated memory location, and if needed initializes the memory with the specified value.

It should also noted that the compiler will allocate space on the stack starting at the next memory location:

In the above example myDouble was created on the stack at memory address 0x1505. A double is 8 bytes long so it will take up 8 consecutive memory locations. Next myChar was created and is one byte long. It resides at 0x1504. Finally myInt is created and is locatated at 0x1500. An int is 4 bytes long so it takes up locations 0x1500 to 0x1503

As was stated in the introduction there are two operators that you need to know about to use pointers

  • & - address of operator. It is the same operator that is used with references
  • * - de-reference operator (It may be easier to think of it as the contents of operator). You can use the & operator to get the address of each of the variables. So the following code will output the addresses of where the variables are located in memory:
//output the address of myInt which is 0x1500
cout << &myInt << endl;

//output the address of myChar which is 0x1504
cout << &myChar << endl;

// output the address of myDouble which is 0x1505
cout << &myDouble << endl

C++ has a pointer variable that makes the use of addresses useful. Remember, a pointer is nothing more than a variable that holds the address of another variable.

Initializing a Pointer

You declare a pointer using the * dereference operator. This operator is used two ways. The first is when you want to declare a pointer. The second is when you want to get the value the pointer is referring to (pointing at). 

in *p;  // Asterisk is used to indicate variable is a pointer

Here a pointer is declared but it has not been initialized. This means that it does not hold a valid address of another variable. As with all variables in C++ an uninitialized pointer contains garbage.

There are basically two ways to initialize a pointer. One is that you use the new operator to allocate memory for the pointer. This will be covered later on. The second is to assign the address of another variable to the pointer

int *ptr;
int x = 3;
ptr = &x

Now ptr holds the address of x and can print or change the value of x indirectly using the dereference operator: 

int *ptr;
int x = 3;
ptr = &x;
    cout << *ptr << endl;

The above says take the memory location that the ptr holds (address of x ) and get the value that is located there. This is done with the dereference operator (*). What prints out is 3.

Lets look at it this way, a pointer is just like any other variable. It sits in memory just like any other variable. The difference is it holds the address of another variable. To get at the value of the variable the pointer is referencing, you use the * operator:

In the above example you will notice that the variable x sits at address 0x500 and that the pointer sits at 0x900. The assignment statement says take the address of the variable x and store it at the location of the pointer ptr (0x900). So an address is stored at 0x900. It is the address of x in this example. The cout statement uses the * before the ptr which means get the contents of the variable it is pointing at. In this case it says get the value that is stored at memory location 0x500 which is 3.

What Prints?

int main()
{
int x = 3;
int *ptr = & x;
    cout << ptr << endl;     // prints  0x500 because this is the address stored in the pointer variable.   
    cout << *ptr << endl;   // prints out 3 because this is what the pointer is referring to (what is stored in x).
    cout << &ptr << endl;  // prints out 0x900 because this is the address where the pointer sits in memory.

These memory locations are used for example purposes only. The real addresses would be much larger hex values.

Pointer Math

A benefit of using pointers is that you can use arithmetic operators on them to move the pointer to other memory locaitons:

ptr++;  // move the pointer ahead by one
ptr+=5;  // add 5 to the address of the pointer

It is important to note that using math on a pointer changes the memory address the pointer holds by the number to move by multiplied by the number of bytes data type is

int *ptr;
ptr++;  // Moves the pointer 4 bytes because the int data type is 4 bytes wide

You don't need to worry about this detail because it is handled for you by the C++ compiler. Of course you could always fool the compiler by using a pointer to different data type:


int main()
{	
	int x = 30;
	char *ptr = (char *) &x;
	ptr++;    //Moves the pointer ahead by one byte not 4

	cout << *ptr << endl;
	
	return 0;
}

Here a char pointer is created an then casted so that it will refer to an int. Incrementing the pointer by one only moves it by one byte which means it will be referring to the second byte of the int. While you can do this you will find that there is very rarely an occasion where this is needed.

Pointers and Arrays

You can use a pointer to traverse or examine elements of an array. You simply create the pointer, and set it equal to the first element of the array:

int[] ar = {2,3,5,6,2,7,8,9,0,12,14,15};
int *ptr = ar;         // equivalent to ptr = &ar[0];
for(int i = 0; i < size; i++)
    cout << *ptr++ << endl;

You don't have to start at the beginning of the array. You can start anywhere simply by specifying the address 

int[] ar = {2,3,5,6,2,7,8,9,0,12,14,15};
int *ptr = &ar[2];    // set pointer to the third element of the array

Pointers and Complex Types

There are times when you are going to want to create a pointer to class or a struct type. This is easily done. Just remember that when you want to access members of the class you need to use the -> arrow operator

int main()
{
    Rectangle rect(5, 6);
    Rectangle *r = &rect;

    cout << "The area of the Rectangle is " << r->area() << endl;
    
    return 0;

Introduction

Up to this point all programs created have been able to access only the amount of memory that has been created when the variables were instantiated. This means that variables, arrays and other objects have a fixed size that is determined before the program begins to execute. This type of memory allocation is called static memory allocation because it is fixed in size. But, what if you need a varying amount of memory that can only be determined during the program execution (runtime)? For example, suppose you are logging temperatures to an array and initially you believe you will never need to add any more values than let's say 20. Later on you discover that you need to actually log 30. So now you need to go back, change the program, and rebuild it before using it. If this number later grows to 35 you have to repeat the process.

The solution to this problem is dynamic memory allocation which is very easily accomplished in C++ with two operators.

The new operator

To allocate memory you use the new operator. This operator returns an address to the memory that was requested . The address returned is called an anonymous pointer because it does not have a variable to reference it. It is up to you to supply that reference.

The new operator is easy to use. You simply create a pointer and set it equal to new whatever you need to allocate.

int *p = new int; // allocate memory for a new int and assign the memory to p

Because the new operator returns a memory address you need a pointer to able to access it. You access the memory through the pointer just like you would with any other pointer reference:

*p = 30;
cout << *p << endl;

This may not seem all that exciting to you. You may be thinking that it would be easier to simply declare an int. You would be right about that and it should be noted that there is overhead involved with using dynamic memory allocation. You should use it sparingly.

A very good use for using new to declare memory is with arrays. Using this method you can create an array of any size. Something you can't do with static memory allocation

int main()
{
   int size;

   cout << "How many numbers do you want to create?" << endl;
   cin >> size;

   int *myArray = new int[size];    // Allocate an array of the requested size

   for(int i = 0; i < size; i++)
      myArray[i] = rand() % 101;

   return 0;
}

he compiler in this case reserves a memory block size long and returns an address to the beginning of that block which is assigned to the pointer myArray. Remember when we pass arrays to functions they are passed by pointer. In other words the address of the first element. If you have this address you can easily access the other elements by using pointer math on the array

You may wonder what the difference is between declaring a normal array and assigning memory to a pointer as we have just done. The most important difference is that the size of an array must be a constant value which limits the number of items if can hold at runtime. Dynamic memory allocation on the other hand, allows for the creation of the necessary memory needed while the program is executing. If the memory requirements change then the program can be written to handle this while it is executing. 

Using new With Complex Types

You can use the new operator with just about any type. Many times it is simply more efficient to dynamically create large objects and delete their memory when you are finished with them.

In this example the new operator is used to create instances of the Rectangle class. Each instance is added to a vector:

int main()
{
	vector vRect;

	int length, width;
	int times = 0;

	cout << "How many Rectangles do you want to create?" << endl;
	cin >> times;
	for (int i = 0; i < times; i++)
	{
		cout << "Enter a length and width " << endl;
		cin >> length >> width;
		vRect.push_back(new Rectangle(length, width));
	}

	return 0;
}

Of course if you want to print out the area for each Rectangle that was created you would have to use the arrow -> operator

for (vector<Rectangle *>::iterator p = vRect.begin(); p < vRect.end(); p++)
{
	Rectangle *r = *p;
	cout << r->area() << endl;
}

Checking For Allocation Failure

On rare occasion something may go wrong in your system that will prevent dynamic memory from being created. If the memory cannot be allocated the new operator will return null. It is important for you to check for the null pointer after attempting to allocate memory. Attempting to add data to memory that was not allocated could cause your program to crash. 

int main()
{
    int *myArray;
  
    myArray = new int[5];
    if(myArray == NULL)
    {
          //Error creating memory. Take appropriate measures. 
    }

When dynamic memory cannot be allocated a bad_alloc exception can be thrown. This will be covered in more detail in the unit on exceptions and exception handling.

Cleaning Up Memory With delete

It is important to note that memory created with new is allocated on the heap so it stays with the program through it's lifetime. By default when your program exits all of the dynamic memory that has been allocated gets freed up. Many programs being used today run non-stop so the memory never gets freed. When you are finished using an object that has been dynamically created it is a very good idea to free the memory up. This is accomplished with the delete operator. The following example shows different data types be allocated and deleted:

int *myInt = new int;
myInt = 25;
delete myInt; // Finished with memory, free it up

Rectangle *rect = new Rectangle(5,3);
int theArea = rect->area();
delete theArea;   // Delete the memory allocated for the complex type

int *myArray = new int[10];
for(int i = 0; i < 10; i++)
    myArray[i] = rand() % 100;
delete[] myArray;   // When deleting memory allocated for an array you have to use []

Failure to clean up after memory allocation can lead to memory leaks getting into your code. I have seen poorly written code that simply crashes because of memory leaks.

Memory Leaks

Memory leaks happen when the pointer that refers to dynamic memory is either reassigned or is deleted improperly. Remember new returns an anonymous pointer that you need to create a reference to. If you do not maintain this reference properly you can end up with a memory block that has no reference. This means it cannot be accessed or deleted any more. Without the ability to delete memory it simply eats up the space used for your program. If this behavior is repetitive then your program can eventually crash. To understand what a memory leak is it is best to look some examples.

// Leak 1
char *str;
str = new char[20];
str = new char[30;

delete[] str;

In this example a pointer to a character array is created. It is first assigned to the new memory created by the new char[20] statement. Next the pointer is reassigned to a new block of memory. When the pointer is reassigned the existing memory exists in the system without a pointer to reference it. At the end delete is called but it will only delete what string is currently referring to (char[30]).

Solution:< br/> Call delete to free up the memory before reassigning the pointer to a new block of memory.

Smart Pointers - New to Version 11

Memory leaks are a real problem in C++ because they are easily put into code by mistake. Of course a lot of this can be because there are programmers who do not understand the proper usage of pointers and memory allocation. Because of the problems associated with memory leaks smart pointers were added to the language

A smart pointer is a class type that acts like a built-in pointer. Because a smart pointer is a complex type it was designed with the ability to handle it's own memory cleanup. This means that when you use a smart pointer you do not need to worry about using delete to free the memory. The smart pointer will free up the memory for you. The smart pointer was implemented in such a way that they can be used anywhere you would use a C++ primitive pointer.

Types of Smart Pointers

Before going any further I should correct some terminology. I use the word primitive pointer because it is built into the C++ language. The correct terminology when dealing with smart pointers is to call them raw pointers. In any event, a raw pointer is a primitive pointer in my mind

There are three different smart pointers that are available for you to use

  1. unique_ptr - Allows exactly one owner for the underlying pointer. The pointer can me moved to a new owner but it cannot be copied or shared.
  2. shared_ptr - This is a reference counted pointer. Which means it contains a reference counter that gets increased each time you assign it. Shared pointers are handy when dealing with container types like the vector. You can wrap elements from the vector in a shared_ptr and then copy it to other containers. The underlying memory is maintained as long as it is needed.
  3. weak_ptr - Used in conjunction with a shared_ptr. A weak_ptr gives access to an that object that is owned by a shared_ptr. It does not increment the reference counter. You can use this pointer when you simply want to examine an object but not modify the reference count.

Using Smart Pointers

In Order to use a smart pointer you must first include . This is part of the std namesapce

Let's start with a simple example that shows how to create smart pointer to create a new instance of a Rectangle:

#include 
#include 
#include "Rectangle.h"

using namespace std;

int main()
{
	int length, width;

	cout << "Enter the length and width for the Rectangle" << endl;
	cin >> length >> width;

	unique_ptr r1(new Rectangle(length, width));

	cout << r1->area() << endl;
        return 0;

Take note of the syntax. An instance of a unique_ptr is being created called r1. An instance of a new Rectangle is being created in it's constructor.

Here is a more complex example. The following code is a modification of the code in a previous section. The code fills a vector with dynamically allocated Rectangles using a smart pointer. After this code executes a loop prints out all of the Rectangles added to the vector. Again this is done using a smart pointer

	}
	
	return 0;
}

Static Class Variables

When you create an instance of a class it becomes an object. All objects have their own instances of class variables. There is an exception to this and that is the static class variable. A static class variable is shared among instances of the class. You declare it like any other variable of a class except that you put the static keyword in front of it.

As an example suppose you want to know how many instances of a class have been created. You could put a static variable in the class and increment it in the constructor. Then each time an instance of the class is created the counter gets incremented. This will allow you to keep count of how many instances of the class have been created.

Here is the class definition for the Rectangle again 

class Rectangle
{
private:	
	int length;
	int width;

public:
        static int count;    // Create the static count variable. Keeps track of instances. 
	Rectangle(int length, int width);
	Rectangle() :length(0), width(0) {}
        ~Rectangle(); // Destructor
	int area();

	void setValues(int length, int width);  //Mutator
	int getLength();   //Accessor
	int getWidth();    //Accessor

	Rectangle add(const Rectangle &r);
	Rectangle operator +(const Rectangle &r);
};

Once you define a static data member, it exists even though no objects of it's class exist. In the above example, no objects of class Rectangle exist even though the static data member Rectangle::count has been defined.

Static member variables of a class in namespace scope have external linkage. The initializer for a static data member is in the scope of the class declaring the member. Because of this you need to initialize the variable in global scope. This is typically done inside the .cpp file where the functions for the classes are created. 

int Rectangle::count = 0;   //Initialize the static variable

Rectangle::Rectangle(int length, int width)
{
	this->setValues(length, width);
	Rectangle::count++;  //Incrment the counter
}
Rectangle::~Rectangle()
{
	Rectangle::count--; // Decrement the counter
}

Because the static variable has external linkage the scope resolution operator is used to specify what class the variable belongs to. Also notice that the data type of the variable needs to be specified.

To access the variable you use the scope resolution operator. In the above code each time an instance of a Rectangle is created the count variable is incremented. Each time an instance is removed from memory the destructor is called which decrements the count. This provides a simple method for keeping track of the number of instances that have been created.

The following shows two instances of the rectangle class being created. After the objects have been created the count variable is printed to the screen. Notice again that because the count variable is static and has external linkage the scope resolution operator is used to specify which class the variable belongs to. When delete is used on one of the Rectangle pointers the destructor is called which decrements the count variable:

int main()
{	
	
	Rectangle *r1 = new Rectangle(4, 5);
	Rectangle *r2 = new Rectangle(6, 7);

	cout << "There are " << Rectangle::count << " Rectangles" << endl;
	delete r1;
	cout << "There are " << Rectangle::count <<  " Rectangles" << endl;
	
	return 0;
}

Introduction

Design patterns have been around for a long time in programming. They first gained traction in 1994 when the book Design Patterns: Elements of Reusable Object Oriented Software was published. A design pattern is way or an approach to solving common software design problems. The book introduced 23 patterns that can be used to solve several generic problems.

Design patters can be classified in one of the following three categories:

  1. Creational Patterns
  2. Structural Patterns
  3. Behavioral Patterns

The breadth and scope of design patterns make it difficult to cover as a section in one class. For this reason this section simply serves as an introduction to design patters. We will look at a couple of different ones

The Singleton Design Pattern

Many times in programming you want to have the ability to use an instance of an object without having to place it in global scope. This is where the Singleton design pattern shines. As an example imagine a menu class that has functionality in it that you would like to have access to from many different functions. One solution is that you could pass the instance of the menu to the function as a parameter. Many times though this is not possible. Let's take a look at the menu class which you may already be familiar with.

The way the functions are setup there is no way to easily pass the menu to them through the parameter list. The solution here is to create the instance of the menu in global scope. Of course we know that this is real bad coding practice. To recap, we want to have the ability to create one instance of the menu and also have the ability to access the instance in multiple functions without using global variables and without passing it through the parameter list.

The best solution to solve the problem is to use the Singleton design pattern. The Singleton design pattern uses a private static pointer to the class that it is contained within:

class Menu
	{
	private:
		vector mi;
		int count;
		void runSelection();
		static Menu *pInstance;   // The static instance pointer
	public:
		Menu();
		void addMenu(string Descript, void(*f)());
		void runMenu();
		void waitKey();
		static Menu *Instance();
	};

This pointer can then be initialized and returned from a function. In this way a pointer to the current instance of the menu can be gotten through a call to an accessor function. Because the pointer is static it must be initialized outside of the class:

//Initialize the static instance pointer
Menu *Menu::pInstance = NULL; 
//Create the accessor function which will return the instance 
Menu *Menu::Instance()
{
    if(pInstance == NULL)
       pInstance = new Menu;

    return pInstance;
}

The first time the Instance function is called pInstance will be NULL and the pointer will be initialized by calling new. For each successive call to the Instance function the pointer will NOT be NULL and the instance will simply be returned.

This allows us to improve the code by removing the global variable. The following shows how the Instance function is called and used.

//Menu removed from global scope
{
   Menu *m = Menu::Instance();   // First time called new will create a new instance of the class
   m.addMenu("1. Function 1", func1);
   m.addMenu("2. Function 2", func2);
   m.addMenu("4. Exit", Exit);
   m.runMenu();
   return 0;
 }
 void func1()
 {
    cout << "Hello from function 1" << endl;
     Menu *m = Menu::Instance(); // Instance pointer is returned
    m.waitKey();
 }
void func2()
{
    cout << "Hello from function 2 " << endl;
    Menu *m = Menu::Instance(); // Instance pointer is returned
    m.waitKey();
}
void Exit()
{
    cout << "Goodbye" << endl;
     Menu *m = Menu::Instance(); // Instance pointer is returned 
    exit(0);
}

Factory Design Pattern

The factory design pattern consists of a static member function that is used to create an instance of the object. This will localize the creation of the object and prevent disturbance of the system when a new type is introduced.

The following example shows a simplistic view of the factory design pattern

 class Rectangle
{
private:
	static int count;  // static count variable
	int length;
	int width;

public:
	static void printCount();  // Accessor for static count

	Rectangle(int length, int width);
	Rectangle() :length(0), width(0) {}
	~Rectangle();
	int area();

	void setValues(int length, int width);  //Mutator
	int getLength();   //Accessor
	int getWidth();    //Accessor

	Rectangle add(const Rectangle &r);
	Rectangle operator +(const Rectangle &r);

	// Factory method to create a Rectangle
	static Rectangle *create(int length, int width);
};   
//Factory method creates a Rectangle
Rectangle *Rectangle::create(int length, int width)
{
	return new Rectangle(length, width);
}

Now the static function can be called to create an instance of the object

int main()
{	
	
	Rectangle *r1 = Rectangle::create(4, 5);
	Rectangle *r2 = Rectangle::create(6, 7);

	Rectangle::printCount();
	delete r1;
	Rectangle::printCount();
	
	return 0;
}
